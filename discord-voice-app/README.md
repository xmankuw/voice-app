# 🎧 غرفة صوتية (نسخة Discord مصغّرة)

تطبيق دردشة **صوتية + مرئية + بث شاشة + دردشة كتابية**، مع **مؤشر يبيّن زمن الاستجابة (ping) بالمللي ثانية**. مبني بأقل تأخير ممكن باستخدام تقنية **LiveKit** (نفس النوع اللي تستخدمه التطبيقات الكبيرة).

> أنت ما راح تكتب أي كود. بس تتبع الخطوات بالترتيب. ياخذ منك تقريبًا **٣٠–٤٥ دقيقة**.

---

## شنو بتحتاج؟
- جهاز كمبيوتر ومتصفح (Chrome مفضّل).
- بريد إلكتروني (لفتح حسابات مجانية).
- صبر شوي — كل الخطوات مجانية وما تحتاج بطاقة.

الحسابات المجانية اللي بنفتحها:
1. **LiveKit Cloud** — يحرّك الصوت والفيديو (الجزء الصعب).
2. **GitHub** — نحط فيه ملفات المشروع.
3. **Vercel** — ينشر الموقع على الإنترنت.

---

## الخطوة ١: حساب LiveKit (الصوت)
1. روح [livekit.io](https://livekit.io) واضغط **Start Building** / **Sign up** (سجّل بحساب Google أو إيميلك).
2. أنشئ **Project** جديد، واختر **Region** الأقرب لك (للكويت/الخليج اختر أوروبا أو الشرق الأوسط إذا متوفر — هذا يقلّل التأخير).
3. ادخل على **Settings → Keys** وأنشئ مفتاح (Create Key). بتطلع لك ٣ قيم، احفظها بمكان آمن:
   - **API Key**
   - **API Secret**
   - **WebSocket URL** (يبدأ بـ `wss://...livekit.cloud`)

---

## الخطوة ٢: ارفع المشروع على GitHub
1. روح [github.com](https://github.com) وافتح حساب (مجاني).
2. اضغط **New repository** → سمّه مثلاً `voice-app` → خلّه **Public** → **Create repository**.
3. في الصفحة الجديدة اضغط الرابط **uploading an existing file**.
4. **اسحب كل ملفات هذا المجلد** (اللي حمّلته من Claude) وأفلتها في الصفحة.
   - تأكد إنك ترفع المجلدات `app` و `components` والملفات `package.json` و `next.config.js` ... إلخ.
   - **لا ترفع** مجلد `node_modules` (أصلاً مو موجود، زين).
5. اضغط **Commit changes**.

---

## الخطوة ٣: انشر على Vercel
1. روح [vercel.com](https://vercel.com) واضغط **Sign Up** بحساب **GitHub** (هذا يربطهم تلقائيًا).
2. اضغط **Add New → Project**، بتلقى مستودع `voice-app` → اضغط **Import**.
3. قبل ما تضغط Deploy، افتح قسم **Environment Variables** وضِف الثلاثة (من الخطوة ١):

   | الاسم (Name) | القيمة (Value) |
   |---|---|
   | `LIVEKIT_API_KEY` | الـ API Key |
   | `LIVEKIT_API_SECRET` | الـ API Secret |
   | `NEXT_PUBLIC_LIVEKIT_URL` | رابط الـ wss:// |

4. اضغط **Deploy** وانتظر دقيقة–دقيقتين.
5. بيطلع لك رابط مثل `https://voice-app-xxxx.vercel.app` — **هذا موقعك!** 🎉

---

## الخطوة ٤: استخدمه مع أصدقائك
1. افتح الرابط، اكتب **اسمك** و**اسم غرفة** (أي اسم، مثلاً `شباب-السبت`).
2. اضغط دخول وسمح للمتصفح يستخدم المايك.
3. **أرسل نفس الرابط لأصدقائك**، وخلّهم يكتبون **نفس اسم الغرفة** → بتلاقون بعض في نفس المكالمة.

---

## وين يبين التأخير؟ 📶
فوق في منتصف الشاشة بتلقى مؤشر زمن الاستجابة:
- **أخضر (أقل من ٨٠ms)** = ممتاز
- **أصفر (٨٠–٢٠٠ms)** = جيد
- **أحمر (أكثر من ٢٠٠ms)** = مرتفع

كل ما الرقم أقل، كان الصوت أسرع وأوضح.

### نصائح لأقل تأخير وأفضل استجابة
- اختر **Region** قريب منك في LiveKit (الخطوة ١).
- استخدم نت **واي فاي قوي** أو كيبل، وابتعد عن النت الضعيف.
- سكّر التطبيقات اللي تستهلك النت بالخلفية (تحميل، تحديثات).

---

## (اختياري) تشغيل على جهازك للتجربة قبل النشر
إذا حاب تجرّبه محليًا:
1. ثبّت **Node.js** من [nodejs.org](https://nodejs.org) (نسخة LTS).
2. افتح المجلد في **Terminal** ونفّذ:
   ```bash
   npm install
   ```
3. انسخ ملف `.env.example` وسمّه `.env.local`، وحطّ فيه مفاتيحك.
4. شغّل:
   ```bash
   npm run dev
   ```
5. افتح المتصفح على `http://localhost:3000`.

> ملاحظة: المايك والكاميرا يشتغلون فقط على `localhost` أو على رابط **https** (مثل رابط Vercel). هذا شرط أمان من المتصفحات.

## لو صار خطأ؟
ارجع لي وانسخ لي رسالة الخطأ اللي تطلع، وأنا أساعدك أحلّها خطوة بخطوة. 🙌

أكثر الأخطاء شيوعًا:
- **ما يشتغل الصوت / يطلع خطأ توكن** → غالبًا أحد المفاتيح الثلاثة غلط أو ناقص في Vercel. راجعها وأعد النشر (Redeploy).
- **ما يطلب المايك** → تأكد إنك فاتح الرابط بـ https (رابط Vercel) أو localhost.
